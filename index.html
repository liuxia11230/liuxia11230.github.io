垂直水平居中

1.行内元素

text-aglin:center

2.块元素

margin 穿透或重叠以及解决办法 （解决办法 BFC 块级格式化上下文）

穿透发生在父子元素之间，子元素设置 margin 会影响其父元素
重叠是同级元素在垂直方向上都设置了 margin

解决办法：利用 BFC （首先变为 BFC 是一个 block 元素）

BFC:特性
1.BFC 中的元素不会影响到其之外的元素
2.Box 垂直方向的的距离由 margin 决定，同一个 BOX 中同级元素垂直方向上 margin 会发生重叠
3.Box 区域不会与 floatBox 发生重叠
4.BFC 计算高度时，浮动元素的高度也参与计算

BFC:触发条件

  1. float: left / right
  2. position: absolute / fixed
  3. overflow: auto / scroll / hidden
  4. display: inline-block / table-caption / table-cell

移动端适配

1.使用 media 媒体查询 2.使用 flex 弹性布局 3.使用 rem：rem 是根据 html 上的字体大小决定的 1rem=html font-size 的大小； 4.使用 vw : 100vw==屏幕宽度

####清除浮动

1.给父元素添加高度/最后添加一个子元素：设置 clear：both/设置伪元素清除浮动/将父元素变为 BFC（块级格式上下文）：overflow:hidden

css hock

CSS hack 的原理
由于不同的浏览器和浏览器各版本对 CSS 的支持及解析结果不一样，以及 CSS 优先级对浏览器展现效果的影响，我们可以据此针对不同的浏览器情景来应用不同的 CSS。

1.用户输入 url 发生了什么

1.DNS 解析
2.TCP 链接
3.HTTP 请求 4.服务器响应 5.浏览器解析

  1. DNS 解析 a.就可以通过 hosts 文件建立映射 ；b.查找本地的 DNS 缓存；c:DNS 服务其缓存；D：DNS 服务其递归查找。找到返回并且加入缓存
  2. 建立 TCP 链接， 3 次握手就是：你在吗，我在，ok；4 次挥手：我要断了，我看看我传完了没，我没有要传的，ok
  3. 解析 HTML，生成 DOM 树。解析 css,生成 css 规则树（css Rule Tree）将 DOM TREE 和 css Rule Tree 结合，生成渲染树（Render Tree）从根节点开始，计算每一个元素的大小、位置，给出每个节点所应该出现的屏幕精确坐标，从而得到基于渲染树的 布局渲染树（Layout of the render tree）。遍历渲染树，将每个节点用 UI 渲染引擎来绘制，从而将整棵树绘制到页面上，这个步骤叫 绘制渲染树（Painting the render tree）
    在解析过程中，可能会产生回流和重绘：

3 次握手：服务端发送 SYN（同步序列编号）是 TCP/IP 建立连接时使用的握手信号到服务器，服务器接受并使用 ack 包进行确认应答，同时发送 syn 和 ack 包给客户端，客户端收到并发送 ack 包确认

4 次挥手：客户端发送释放谅解的标示信息 FIN 并停止发送数据，服务端接到后发送 ACK 进行确认，此时当还有数据没有传输完的时候，数据会继续传输，当数据传输完了之后，发送 FIN 给客户端，客户端接收 ACK 后会进入一段等待时间，当该时间段没有重新请求，此时就会断开连接

渲染过程碰到 JS 文件怎么处理？
javaScript 的加载、解析和执行会阻塞 DOM 的构建。
也就是说：在构建 DOM 的时候，HTML 解析器如果碰到 JavaScript，那么就会停止构建 DOM，将控制权交给 JavaScript 引擎，等 JavaScript 运行完毕，浏览器再从中断的地方恢复 DOM 构建。

前端性能优化

##构建优化

1.开启 gzip 压缩，gzip 压缩效率非常高。可以达到 70% 2.使用 CDN 减少打包体积

3.预渲染

##感知性能优化

1.骨架屏

2.loading

##资源优化

1.预加载
无线网后资源加载受限与网络。因此资源缓存，减少网络请求

##图片优化

1.懒加载，将真实图片 url 存在 data-url 中，只要 src 没有 url 就不会发出请求，滚动事件算处当前图片是否在可是区域。（当前元素到顶的高度-滚动条的高度<当前屏幕可是区域高度）则显示图片

2.压缩大小，减小带宽

##合理的利用缓存

1.强缓存 catch-control:max-age,public:能被浏览器和代理服务器缓存。就是给资源设置一个过时间，每次
请求都会看时间是否过期

2.强缓存失效会走协商缓存。
设置协商缓存：etag：“”，last-modified:”时间”
为什么要有 etag？
你可能会觉得使用 last-modified 已经足以让浏览器知道本地的缓存副本是否足够新，为什么还需要 etag 呢？HTTP1.1 中 etag 的出现（也就是说，etag 是新增的，为了解决之前只有 If-Modified 的缺点）主要是为了解决几个 last-modified 比较难解决的问题：

一些文件也许会周期性的更改，但是他的内容并不改变(仅仅改变的修改时间)，这个时候我们并不希望客户端认为这个文件被修改了，而重新 get；

某些文件修改非常频繁，比如在秒以下的时间内进行修改，(比方说 1s 内修改了 N 次)，if-modified-since 能检查到的粒度是秒级的，这种修改无法判断(或者说 UNIX 记录 MTIME 只能精确到秒)；

某些服务器不能精确的得到文件的最后修改时间。

Docker

镜像->通过镜像变->容器、仓库

概念

  • 基于go语言实现
  • Docker可以让开发者打包他们的运用以及运用依赖的环境的（node mysql）到一个轻量级、可移植的容器中，
